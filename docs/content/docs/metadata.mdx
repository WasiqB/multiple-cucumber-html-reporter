Multiple Cucumber HTML Reporter can display the browser, version, platform, and device type each feature was executed on.

Metadata can be displayed either in the Features overview table or within each scenario's details.

When metadata is not provided, the reporter automatically adds the local platform name and version, current machine username, device hostname, Node.js version, reporter version, hostname, and CPU architecture. If partial metadata is provided, for example only browser details, the missing standard fields are still filled automatically.

## Adding Metadata

There are two primary ways to add metadata:

### Option 1: Same metadata for all the Features

If you provide metadata object, same metadata will be added to all the features in the report.

```typescript
await generate({
  metadata: {
    browser: {
      name: 'chrome',
      version: '60'
    },
    device: 'Local test machine',
    platform: {
      name: 'ubuntu',
      version: '16.04'
    }
  }
});
```

Any values you provide take precedence over automatic defaults.


<Callout title="Deprecated" type='warning'>
Metadata type of `Metadata` is deprecated and will be removed in the next major version.

Use metadata of `Record<string, Metadata>` type instead.
</Callout>

### Option 2: Feature specific metadata (Recommended)

If your test scenarios are running in different environments, you can define metadata based on the feature file name.

The `metadata` object should be defined as follows:

```typescript
const metadata: { [key: string]: Metadata } = {
  'saucedemo.feature': {  // [!code highlight]
    browser: {
      name: 'chrome',
      version: '148',
    },
  },
  'restful-booker.feature': { // [!code highlight]
    browser: {
      name: 'api',
      version: '',
    },
  },
};

await generate({
  // other options
  metadata,
});
```

<Callout title="Note" type='idea'>
The highlighted lines in the above example is the actual feature file name along with their respective set of Metadata object.
</Callout>

## Supported Metadata Values

To ensure proper icons are displayed, use the following predefined values for certain fields:

### `browser.name`
- `internet explorer`
- `edge`
- `chrome`
- `firefox`
- `safari`
- `api`

### `platform.name`
- `windows`
- `osx`
- `linux`
- `ubuntu`
- `android`
- `ios`

## Custom Metadata

If you need more than just the standard browser and platform information, you can use **Custom Metadata**.

### Configuration

<Callout title="Deprecated" type='warning'>
Metadata type of `Array<{ name: string, value: string }>` and `customMetadata` property is deprecated and will be removed in the next major version.

Use metadata of `Record<string, Metadata>` type instead.
</Callout>

Set `customMetadata: true` in your options:

```javascript
report.generate({
  customMetadata: true,
  metadata: [
    { name: "Environment v.", value: "12.3" },
    { name: "Plugin v.", value: "32.1" },
    { name: "Variable set", value: "Foo" },
  ],
});
```

<Callout title="Caution" type="warn">
Custom metadata will override regular metadata completely. Try to avoid using more than 10 metadata variables, as it can cause layout issues.
</Callout>
